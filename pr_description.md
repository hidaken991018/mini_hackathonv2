# PR Title
feat: Gemini 3 Pro による「完成イメージ」「インフォグラフィック」生成機能の追加と通知UI改善

# 概要
通知機能において、**Gemini 3 Pro** を活用した高度な画像生成機能を実装しました。
1. **2種類の画像生成**: レシピ提案時に「完成イメージ」と「調理工程インフォグラフィック」の2枚を生成します。
2. **通知ビジュアルの強化**: 通知カードと詳細画面それぞれに最適な画像を表示するようにしました。

# 変更点

## バックエンド
- **モデル更新**: 画像生成モデルに `gemini-3-pro-image-preview` を採用しました。
- **画像生成ロジック (`app/api/recipe/notify/route.ts`)**:
    - **完成イメージ (Dish Image)**: シズル感のある料理の完成写真を生成し、`Notification` モデルに紐付けます。
    - **インフォグラフィック (Infographic)**: 調理手順や材料を視覚的に解説する画像を生成し、`Recipe` モデルに紐付けます。
- **GET API (`app/api/notifications/route.ts`)**: インフォグラフィック画像をフロントエンドに返却するようにレスポンスを拡張しました。

## フロントエンド
- **`components/NotificationCard.tsx`**: 通知カードには「完成イメージ」を表示します。
- **`components/RecipeSlideModal.tsx`**:
    - ヘッダー画像の表示を廃止しました。
    - レシピ詳細（本文）の冒頭に「インフォグラフィック画像」を大きく表示するようにレイアウトを変更しました。
- **型定義 (`types/index.ts`)**: `Notification['recipe']` に `imageUrl` プロパティを追加しました。

## その他
- **アセット追加**: 警告用・期限切れ用の静的アイコン画像を追加しました。

# 検証結果
- 検証スクリプト `scripts/verify-notify-image.ts` にて、以下を確認しました。
    - APIが正常（Status 200）に終了すること。
    - 2種類の画像ファイルがローカル（`public/images/dishes/` および `public/images/infographics/`）に生成・保存されること。
    - データベースにそれぞれの画像URLが正しく保存されること。
